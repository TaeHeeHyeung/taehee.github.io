    깃허브 특징
  1. 파일은 스냅샷의 연속으로 구성되어있다.
  2. 깃허브는 모든 명령을 로컬로 실행하기 때문에 히스토리를 조회할때 서버 연결 필요없어 성능에 이점이 있다.
  3. 해시값을 활용하여 무결하다.
  4. 커밋하면 절대 잃어버릴 일이 없다
  5. 세 가지 상태modified staged commit 1. 깃디렉토리: clone 할때 깃디렉토리가 생성됨 2. 워킹트리 : 프로젝트의 특정버전을 checkout한것으로 깃디렉토리 안의 데이터베이스에서 파일을 가져와 워킹트리를 만든다. 3. 스테이지 에어리어:곳 커밋할 파일에 대한 정보를 저장한다. index라고도 부름 깃으로하는일 워킹트리에서 파일을 수정한다.
